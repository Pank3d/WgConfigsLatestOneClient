## Развёртывание в режиме одного экземпляра

Этот файл описывает, как запустить проект в режиме **одного экземпляра `wgProjectServersUtil`** (без `wgProjectServersUtil-1` и `wgProjectServersUtil-2`) с использованием обновлённого `docker-compose.yml`.

### 1. Предварительные требования

- Установлен **Docker** и **Docker Compose**.
- Есть доступ к серверу (или локальной машине), где будет крутиться VPN и бот.
- Подготовлены значения для переменных окружения в файле `wgProjectServersUtil/.env`.

### 2. Структура, которая используется

В режиме одного экземпляра используется корневой `docker-compose.yml` со следующими сервисами:

- `wg-easy` — основной VPN‑сервис (`wgProjectServersUtil`), порты:
  - UDP `800`
  - TCP `500`
- `next-app` — бот / веб-приложение (`wgBot`), порт:
  - HTTP `3000`

### 3. Настройка `.env` для `wg-easy`

1. Перейдите в директорию проекта:

   ```bash
   cd WgManyConfigsLatestOneClient
   ```

2. Откройте или создайте файл `wgProjectServersUtil/.env`.
3. Заполните в нём параметры WireGuard и общие настройки (адрес сервера, ключи, порт и т.д.) согласно вашей инфраструктуре.

> Конкретный набор переменных зависит от реализации `wgProjectServersUtil`. Используйте существующий пример `.env` (если есть) или комментарии внутри файла.

### 4. Запуск в режиме одного экземпляра

Из корня проекта (`WgManyConfigsLatestOneClient`) выполните:

```bash
docker compose up -d --build
```

Это:

- соберёт и запустит контейнер `wg-easy` с конфигом из `wgProjectServersUtil/.env`,
- поднимет контейнер `next-app` (бот / веб‑панель) на порту `3000`.

### 5. Проверка работы

- Убедитесь, что контейнеры запущены:

  ```bash
  docker compose ps
  ```

- Проверьте, что порты открыты снаружи (на сервере должны быть проброшены:
  - UDP `800` и TCP `500` для VPN,
  - TCP `3000` для бота/панели, если нужен внешний доступ).

### 6. Остановка/перезапуск

- Остановить контейнеры:

  ```bash
  docker compose down
  ```

- Перезапустить с пересборкой:

  ```bash
  docker compose down
  docker compose up -d --build
  ```

### 7. Чем этот режим отличается от масштабируемой схемы

- Запускается **только один** экземпляр `wgProjectServersUtil` (`wg-easy`) вместо нескольких (`wg-easy`, `wg-easy-1`, `wg-easy-2`).
- Все клиенты садятся на один WireGuard‑сервер (порты `500`/TCP и `800`/UDP).
- Упрощённая схема подходит для:
  - тестового стенда,
  - небольшого количества пользователей,
  - отладки логики бота и конфигов VPN на одном сервере.
